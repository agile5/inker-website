Responsive Emails.

Inker use Zurb Ink to provide you with responsive css framework, you get all the goodies + a sass component oriented structure.

// Cross compatible button for email services
{% macro button(label='default', link='#', class='', align='left') %}
 <table class="button {{class}}" align="{{align}}">
  <tr>
    <td>
      <a href="{{link}}">{{label}}</a>
    </td>
  </tr>
 </table>
{% endmacro %}

// import component in template
{% from "/html-components/component.button.html" import button %}


//Usage
button('Google', 'http://www.google.com', 'button-green', 'left');
// en_US.yml
btn_accept : Accept
// fr_CA.yml
btn_accept : Accepter

//Usage
{{ btn_accept | trans }}

Tools setup to make you & your team more productive.

Testing emails has always been akwards, with inker you stop loosing time, you have your tools ready for all the testing you wants.

Send to litmus or to your own email address with a simple grunt command. Inker can also live reload your changes at every key stroke.

// Send a specific template to litmus
grunt litmus:dist/output/sidebar_hero/index.html

// Send a template to any email address
grunt email --fileSrc=dist/output/example.html

// Watch changes & reload your templates in your browser
grunt watch

A delivery server to rule them all.

Using a library to send to sendgrid? That's great, but what happen when sendgrid is down?

Inker nodejs server will redirect the request to your prefered failover provider so you stop loosing emails.
